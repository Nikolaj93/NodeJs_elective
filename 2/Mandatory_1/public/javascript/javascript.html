Type coercion

Type coercion is the process of converting a certain value to another, for instance a Number to String. It can also lead to great frustration when JavaScript implicitly handles something in a way that was not meant to be.

Arrow functions

In the beginning of our NodeJS course, we used regular functions to execute some particular code but we quickly changed to use arrow functions instead. There's a specific reason as to why we made the change, but I quite like the look of a arrow function and I think it provides some readibility. There are also some negatives to arrow functions in that it is syntactically anonymous, meaning that arrow functions are not named and that can make it harder to debug to some degree. There's no self referencing either. The main upside and the reason why our course made the switch is because there is no binding of 'this'. In classic functions, 'this' is contextually bound while in arrow functions 'this' is bound to the code that contains the arrow function.
